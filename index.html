    ⚡ Cloudflare Worker URL

    A Cloudflare Worker acts as a CORS proxy for features that can't be called directly from the browser (pollen data, NVD API key authentication, RSS feeds). Deploy the included pollen-proxy-worker.js to your free Cloudflare account and paste the URL here.

    Applies to all features that use the worker — pollen data, NVD/CVE requests, and RSS feeds. Individual widgets can override this with their own URL in their settings.

    Per-widget overrides: The Pollen (in Weather settings), RSS Feed widget, and CVE Bulletin widget each have their own optional Worker URL field. If filled in, that widget uses its own URL instead of this global one. Leave them blank to use this global setting.

    ⚠ Lo‑fi Audio in Microsoft Edge

    Edge blocks audio from YouTube iframes by default due to its strict autoplay policy. Firefox and Chrome don't have this restriction. To fix it in Edge:

    1. Click the link below to open Edge's Media Autoplay settings
    2. Under Allow, click Add
    3. Enter your page's address — e.g. http://localhost:8080 or your NAS IP such as http://10.163.1.236
    4. Click Add, then reload this page

    Note: edge:// links must be opened directly in Edge's address bar — they cannot be clicked from within a web page. Copy the address below and paste it into Edge's address bar.

    edge://settings/content/mediaAutoplay

    Weather Locations

    Tree, grass, and weed pollen levels can be shown on each weather tile. Pollen data requires a paid third-party API subscription — pricing varies by provider. Your key is stored locally and included in exported settings files. Note that some providers may also require a CORS proxy to work from a browser — use the HTTP Proxy setting in the App Monitor section if needed.

    A free Cloudflare Worker can act as a CORS proxy for pollen requests. Deploy the worker script from the Cloudflare Workers dashboard and paste your worker URL above. Worker script available in your settings export.
